# La respuesta corta: depende. La larga, en 5 minutos.

Si estás leyendo esto, probablemente tengas una idea en la cabeza: "necesito una web" o "tengo que mejorar la que tengo". Y al buscar opciones te encuentras con **tres familias**: WordPress, Wix/Webflow y "desarrollo a medida" (que puede ser Next.js, Astro, Laravel, Django…). Cada una tiene sus defensores y sus detractores. La típica pregunta de foro — "¿cuál es mejor?" — no tiene respuesta útil.

La pregunta útil es: **¿qué necesita tu caso concreto?** Porque las tres opciones resuelven problemas distintos con modelos de coste, mantenimiento y velocidad diferentes. Aquí te lo explico sin jerga tribal.

## Cuándo WordPress sigue siendo la elección correcta

WordPress lleva 20 años dominando el mercado por algo. Hay escenarios donde sigue siendo la opción más sensata:

- **Blog o web personal/de empresa con menos de 10 páginas.** Si tu web es básicamente texto, fotos y un formulario de contacto, WordPress + un tema bien elegido te da algo presentable en una semana.
- **Presupuesto ajustado y sin conocimientos técnicos.** El ecosistema de temas (gratuitos y de pago) hace que una web decente se monte por menos de lo que cuesta un dominio custom. Y si te atascas, hay miles de freelancers que conocen WordPress.
- **Necesitas que alguien externo la mantenga.** Si contratas a un estudio pequeño o un freelance WordPress para mantenimiento mensual, les estás dando un campo donde son eficientes. Estándar = reemplazable.
- **Integraciones estándar (newsletter, analytics, SEO plugins).** WordPress tiene plugin para casi todo. Si tus necesidades son convencionales, no hace falta reinventar.

## Cuándo WordPress se queda corto

Hay escenarios donde WordPress se convierte en un problema antes de empezar:

- **Necesitas login de usuarios, áreas privadas, dashboards.** WordPress lo hace con plugins y temas, pero se vuelve frágil y lento rápido. Cada 5-10 plugins pesados compiten por recursos.
- **Ecommerce con lógica compleja.** Si vendes productos configurables, suscripciones, multi-almacén o integraciones con tu ERP/CRM, WooCommerce se queda corto. Magento/Adobe Commerce es serio pero caro y pesado.
- **Integraciones a medida con APIs externas.** Sincronizar tu web con tu sistema de inventario, CRM, o automatizaciones internas: cada integración es un plugin a mantener.
- **Necesitas rendimiento alto (Core Web Vitals en verde, primera carga <2s).** WordPress requiere mucho trabajo de optimización (caché, imágenes WebP, CSS crítico, hosting dedicado) para llegar donde Next.js o Astro llegan por defecto.
- **Multi-idioma serio.** WordPress lo hace con WPML o Polylang, pero traduire 50 páginas con campos custom, productos variables y SEO por idioma es frágil.
- **Tu web NO es tu sitio estático; es tu producto.** Si tu modelo de negocio es una plataforma o app (marketplace, SaaS, dashboard), WordPress es el anti-patrón.

Si encajas aquí, necesitas desarrollo a medida.

| **Coste inicial**     | 600-3.000 € (tema + contenido + puesta en marcha)                                        | 1.200-6.000 €+ (más si es plataforma)                |

## La regla práctica

Tres escenarios, respuesta binaria:

- **Tienes un blog personal o una web de empresa con <10 páginas y un formulario de contacto.** → **WordPress**. No necesitas más.
- **Vendes online con catálogo cambiante (pocos productos, lógica estándar, Stripe/PayPal).** → **Woocommerce** (si quieres Open Source) o Shopify (si quieres olvidarte). Cualquiera de los dos funciona.
- **Necesitas login de usuarios, integraciones a medida (ERP, CRM, APIs internas), multi-idioma serio, o tu web es tu producto.** → **Desarrollo a medida**. Aquí WordPress te va a hacer perder tiempo y dinero en mantener una herramienta inadecuada.
