## Empezar de cero, otra vez, no

Construir componentes para Claude Code —agentes, skills, comandos, hooks, plugins, servidores MCP— tiene un patrón que se repite: la misma estructura, los mismos errores, la misma validación olvidada. Después de copiar y pegar el mismo andamiaje varias veces, lo convertí en un ecosistema de plantillas.

> Una plantilla no es solo un punto de partida: es una decisión de diseño congelada para que no tengas que volver a tomarla mal.

## Ocho tipos de módulo, un patrón

El sistema cubre **ocho tipos de módulo**: agentes, skills, comandos, hooks, plugins, servidores MCP, configuraciones `.claude/` y repositorios GitHub. Cada uno sigue el mismo patrón de tres piezas:

- **Plantilla** (`plantilla_X/`): el playbook didáctico.
- **Ejemplo** (`ejemplo_X/`): una referencia funcional que de verdad corre.
- **Validador** (`validar_X.py`): comprobación automática, con modo `--strict`.

## Validación como red de seguridad

Un `BaseValidator` reutilizable centraliza la lógica; cada módulo añade lo suyo. Con pre-commit hooks para lint de YAML/JSON y detección de placeholders, y workflows de GitHub Actions independientes, el repo se mantiene `claude-init`-ready: **8/8 módulos validando en verde**.

## Estado

El proyecto va por la **v1.0.0**, es **open source (MIT)** y Python 3.12+. Su objetivo es simple: que construir algo para Claude Code empiece en «rellena los huecos», no en «página en blanco».
